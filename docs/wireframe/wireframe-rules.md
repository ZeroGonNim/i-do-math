> 레이아웃 규격·컴포넌트 스펙: [wireframe-ref.md](./wireframe-ref.md) | 검수: [wireframe-checklist.md](./wireframe-checklist.md)

## §1. 실행 절차

### 디바이스 판단

| 조건 | 대상 디바이스 |
|------|-------------|
| 디바이스 **명시** | 명시된 디바이스만 |
| "반응형" / "전체" | DT, LT, TB, MB 전부 |
| APP / 모바일 앱 | **MB 단독** (태블릿 명시 시 TB 추가) |
| 웹 / 관리자 | DT, LT (모바일 명시 시 TB, MB 추가) |
| 판단 불가 | **사용자 확인 후 진행** |

> 근거: 기획서 상단 제목·용도에서 "APP", "사용자(APP)", "관리자(WEB)" 등 키워드 확인.

### 작업 순서

1. **화면 스펙 읽기 (인덱스 기반)**
   - `screen-index.md`가 있으면 → 인덱스에서 대상 화면의 **시작줄/줄수** 확인 → `Read(기획서, offset=시작줄, limit=줄수)`로 해당 화면만 읽기
   - 인덱스 없으면 → 기획서 전체 읽기
   - ⚠️ 기획서 전체를 읽지 말 것. 대상 화면 섹션만 읽어야 토큰 절약 + 속도 향상.
2. 대상 디바이스 결정
3. **보드 목록 추출 (필수 선행)**
   - **Step A**: ASCII 다이어그램 **상단 첫 줄**의 `▶` 항목을 **모두** 열거
   - **Step B (▶ 폴백)**: `▶` 없는 화면 → 기본 보드 1개(화면 ID 사용) + 팝업/alert/오버레이/바텀시트 키워드 → 추가 보드 / 단순 상태 설명("~시 노출") → 기본 보드 포함
   - **예외**: 다이어그램 **본문 내부** `▶`(예: `▶ 5-1 본인확인서비스`)는 외부 참조. 보드 대상 아님.
   - 보드 목록 명시 (예시):
     ```
     [ ] {화면ID}-{디바이스}       → 메인 화면
     [ ] {화면ID}-{디바이스}-popup → 팝업 화면
     ```
4. 스펙값 **그대로 사용** (요약·재작성 금지)
5. 기존 동일 보드 삭제 → 신규 생성
6. 디바이스별 순서: DT/LT→GNB→사이드바→콘텐츠→풋터 / TB/MB→상태바→헤더→콘텐츠→FAB→탭바
7. 배지→설명패널 순서
8. **1회 실행 = 1화면** (타임아웃 방지)
9. **경량 검수**: [wireframe-checklist.md](./wireframe-checklist.md) 수행

> ⚠️ 보드 목록 확정 없이 작업 시작 금지.

## §2. ⛔ HARD RULE: 텍스트 생성 3단계

> **모든 텍스트**에 예외 없이 적용. resize_node 생략 → 세로 1글자 표시 치명적 결함.

```
① create_text  → 정렬 파라미터 필수
② resize_node  → 너비 고정 (⛔ 절대 생략 금지)
③ move_node    → y좌표 배치
```

### 너비 참조표

| 컨텍스트 | 너비(px) | 산출 |
|---------|---------|------|
| MB 보드 전체 | **358** | 390-16×2 |
| TB 보드 전체 | **720** | 768-24×2 |
| LT 보드 전체 | **960** | 1024-32×2 |
| DT 보드 전체 | **1200** | 1440-120×2 |
| 버튼 내 텍스트 | **버튼w** | 동일 너비 → CENTER 자동 |
| Modal 본문 (MB/TB/LT/DT) | 318/352/384/416 | 모달w-패딩×2 |
| DESC 본문 | **392** | 440-24×2 |
| DESC 불릿 (·) | **364** | 392-28 |
| DESC 서브항목 | **352** | 392-40 |
| 팝업 내 본문 | 팝업w-패딩×2 | 계산 |

> 테이블에 없는 경우: `부모 너비 - 좌우패딩 합계`

### 수정/검수 시에도 3단계 전부

```
❌ 삭제 → create_text만             ✅ 삭제 → create_text → resize_node → move_node
```

## §3. 좌표 기준

| 상황 | 기준 |
|------|------|
| 페이지 루트에 노드 생성 | 캔버스 절대좌표 |

> ⚠️ `move_node`는 항상 로컬좌표. 절대좌표 사용 시 프레임 밖으로 사라짐.

## §4. 배지 규칙

- Description **명시 항목에만** 배지 생성. 서브항목(2-1 등)·※주석은 제외.
- 크기: 22×22 원형 red, 번호 11px Bold white CENTER
- 위치: `x=board.x-14`, `y=board.y+(comp.y+comp.h/2)-11`
- **번호 = Description 원본 번호** (점프 허용, 재부여 금지)

### 팝업 보드 분리

| 키워드 | 처리 |
|--------|------|
| 팝업/alert/오버레이/바텀시트/모달 | **별도 보드** + 독립 번호 재부여 |
| 토스트 | 기본 보드 포함, 배지 불필요 |
| ~시 노출/~문구 표시 | 기본 보드 포함 |

### 필수: 매핑 테이블

```
| 배지# | Description항목 | 대상보드 | 컴포넌트 | comp.y | comp.h | 배지y |
```

## §5. ⛔ HARD RULE: 설명 패널 (DESC)

### 크기/위치

| 디바이스 | x | 크기 |
|---------|---|------|
| DT | board.x+1480 | 520×900 |
| LT | board.x+1060 | 480×768 |
| TB | board.x+800 | 460×1024 |
| MB | board.x+422 | 440×844 |

fill=#FFF, stroke=#E0E0E0 1px, r=4

### 텍스트 너비/x

| 요소 | 너비 | x(로컬) |
|------|------|---------|
| 메타 라벨/값 | 84/308 | 24/108 |
| Description 헤더·번호항목 | 392 | 24 |
| 불릿(·) / ※주석 | 364 | 28 |
| 서브항목(N-1.) | 352 | 40 |

### [CSS] 힌트 처리

기획서 Description에 `[CSS]` 힌트 줄이 있으면:
- `create_text`로 **그대로** 포함 (불릿 없이, 회색 `#888` 14px Regular)
- 삭제·생략 금지 — 코드 생성 AI가 구현 구조를 읽는다

```
예시:
ㆍ캐릭터 이미지 우측 배치
[CSS] relative 컨테이너 / 캐릭터: absolute right:0 bottom:0
```

### ⛔ Description 텍스트 처리 (HARD RULE)

> Description 항목은 **기획서 원문을 그대로 복사**. 추론·요약·의역·재작성 절대 금지.

```
1. 기획서에서 Description 블록을 통째로 추출 (번호·불릿·서브항목 구조 유지)
2. 추출한 텍스트를 변형 없이 create_text에 그대로 전달
3. 에이전트가 하는 것은 레이아웃(y좌표, 너비, 폰트)만 — 텍스트 내용은 건드리지 않음
```

```
❌ 기획서: "· 클릭 시 메인 화면으로 이동"  →  에이전트: "· 메인으로 이동하는 기능"
✅ 기획서: "· 클릭 시 메인 화면으로 이동"  →  에이전트: "· 클릭 시 메인 화면으로 이동"
```

### 메타 영역 프리셋 (고정 좌표)

> 메타 5행은 항상 동일 구조. 아래 좌표를 연산 없이 그대로 사용.

| 행 | 라벨 | y | 라벨x | 값x |
|----|------|---|-------|-----|
| 1 | 화면ID | 24 | 24 | 108 |
| 2 | 경로 | 60 | 24 | 108 |
| 3 | 설명 | 96 | 24 | 108 |
| 4 | 개발유형 | 132 | 24 | 108 |
| 5 | 디바이스 | 168 | 24 | 108 |
| — | 구분선 | 196 | — | w=392 |
| — | Description 헤더 | 213 | 24 | — |
| — | **첫 항목 시작** | **249** | 24 | — |

라벨: 14px Bold, w=84 / 값: 14px Regular blue, w=308
구분선: x=24, h=1, fill=#E0E0E0

### 콘텐츠 구조

1. **메타**: 위 프리셋 테이블 그대로 사용
2. **구분선**: y=196, h=1
3. **Responsive Notes** (DT/LT 전용): y=213 위치에 삽입, 이후 항목 y를 그만큼 밀기
4. **Description 헤더**: "Description(화면설명)" 14px Bold, y=213
5. **항목**: `{번호}. {컴포넌트명}` 13px Bold + `· {불릿}` 12px #444, 첫 항목 y=249

### 항목 배치 순서

```
N. 번호 항목 제목
  N-1. 서브항목
  N-2. 서브항목
  · 불릿 텍스트
```

### ⛔ y좌표 계산 (HARD RULE)

**공식**: `nextItem.y = prevItem.y + prevItem.height + gap`

- 간격: 같은 그룹 내 **4px** / 그룹 간(N→N+1) **16px**
- 텍스트 높이: **16×N줄+4** (1줄=20, 2줄=36, 3줄=52)

**첫 항목부터 계산** (메타는 프리셋 사용):
```
item[0].y = 249
이후: 같은 그룹 → prev.y+prev.h+4 / 새 그룹 → prev.y+prev.h+16
```

**실전 예시**:
```
1. "1. 페이지 타이틀 영역"      y=249  h=20
2. "2. 주요 정보 표시"          y=285  h=20   (249+20+16)
   "· 항목A, 항목B 등..."      y=309  h=36   (285+20+4, 2줄)
   "2-1. 세부 항목"             y=349  h=20   (309+36+4)
   "· 세부 설명 텍스트..."     y=373  h=20   (349+20+4)
3. "3. 액션 버튼"               y=409  h=20   (373+20+16)
```

> ⛔ 동일 간격(매 +24px) 배치 금지. ⛔ y좌표 눈대중 금지.

### cascade 규칙

항목 수정 시 그 아래 **모든 항목** y좌표 재계산.
```
❌ 3번만 수정, 4·5·6번 방치    ✅ 3번 수정 → 4→5→6 전부 재계산
```

### 팝업 전용 보드

분리 시 독립 번호(1,2,3...) 재부여. DESC 화면ID: `{원본ID} (팝업)`

### 불릿 작성

패턴: `[보이는 것] + [동작] + [결과]`. 접두어: `[공통]`/`[DT]`/`[MB]` 등. 원문 그대로 사용.

## §6. 네이밍

```
보드: {ID}-{디바이스}        → MI1-MB
패널: {ID}-{디바이스}-DESC   → MI1-MB-DESC
배지: badge-{#}-{디바이스}   → badge-1-MB
컴포넌트: {디바이스}-{이름}  → MB-GNB
```
