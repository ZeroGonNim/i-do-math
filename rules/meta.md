# Meta Rules: 하네스 자가 최적화 규칙

> 이 규칙은 하네스(템플릿) 자체의 효율성을 극대화하기 위한 **상위 지침**이다.
> 작업 분류(T0/T1/T2)와 워크플로우는 `AGENTS.md`가 canonical이다. 본 문서는 중복하지 않는다.

---

## 1. JIT(Just-In-Time) 컨텍스트 (Token/Cost 상쇄)

방대한 규칙을 매번 읽지 않고, 필요한 시점에만 로드한다.

- **Selective Reading**: `CLAUDE.md` §3 라우팅 테이블에 정의된 파일만 `read_file` 한다. 연관 없는 파일은 무시한다.
- **Search First**: 특정 규칙이 기억나지 않으면 전체를 읽기 전 `grep_search`로 키워드만 검색하여 컨텍스트를 보충한다.

---

## 2. 기술 스택 자동 적응 (Rigidity 상쇄)

프로젝트 환경에 따라 규칙을 유연하게 변조한다.

- **Stack Detection**: 세션 시작 시 `package.json` 또는 환경 파일을 읽어 현재 스택을 확정한다.
- **Rule Adaptation**: 확정된 스택에 따라 공통 규칙(TypeScript 등)의 세부 사항을 해당 프레임워크(Next.js, Vite 등)에 맞게 해석하여 적용한다.

---

## 3. 에이전트 코칭 (Learning Curve 상쇄)

사용자가 시스템을 배우게 하지 말고, 에이전트가 가이드를 제공한다.

- **Why-Explain**: 복잡한 절차(작업 지시서 승인 등)를 요구할 때, "왜 이 절차가 필요한지"와 "이로 인해 얻는 이득(예: 할루시네이션 방지)"을 1줄로 설명한다.
- **Proactive Guide**: 사용자의 요청이 모호할 경우, 하네스 규칙에 근거하여 "최선의 요청 방식"을 예시와 함께 제안한다.

---

## 4. 규칙 자가 최적화 (Maintenance 상쇄)

하네스의 낡은 규칙을 AI가 스스로 찾아내고 개선 제안을 한다.

- **Friction Logging**: 작업 중 규칙이 실제 코드 구현에 방해가 되거나(Outdated), 불필요한 오버헤드를 발생시킨 경우 메모리(`save_memory`) 또는 특정 로그 파일에 기록한다.
- **Auto-Refining**: 분기별 또는 대규모 작업 완료 후, 기록된 '마찰 지점'을 분석하여 `CLAUDE.md`나 `rules/` 수정안을 PR 형태로 사용자에게 제안한다.
